# Contributing

Thanks for taking the time to look at this project. This is a **personal portfolio website**, not an open source product — but contributions in the form of bug reports, suggestions, and fixes are genuinely welcome.

---

## What I welcome

| Type | Examples |
| ---- | -------- |
| 🐛 Bug reports | Broken layout, missing images, JS errors, accessibility issues |
| 💡 Suggestions | UX improvements, performance ideas, SEO tips |
| 🔧 Small fixes | Typos, broken links, outdated content |
| ♿ Accessibility | Contrast issues, screen reader problems, keyboard navigation |

## What I don't need

- Full redesigns or style changes (this is intentionally personal)
- New features or sections (I manage content myself)
- Dependency additions

---

## Reporting a bug

1. [Open an issue](https://github.com/MrFabry/mrfabry.github.io/issues/new)
2. Include browser + OS + screen size if it's a visual bug
3. A screenshot helps a lot

---

## Submitting a fix

1. Fork the repo
2. Create a branch: `git checkout -b fix/short-description`
3. Make your change — keep it focused and minimal
4. Test it locally by opening `index.html` in a browser
5. Open a Pull Request with a clear description of what you changed and why

Please keep PRs small. One fix per PR is ideal.

---

## Code style

- **HTML** — semantic, indented with 4 spaces
- **CSS** — variables defined in `:root`, mobile-first where possible
- **JS** — vanilla, no frameworks, no build step required
